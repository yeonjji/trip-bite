# 01-PRD: Product Requirements Document

> **프로젝트**: 여행한입 (Trip Bite)
> **작성일**: 2026-03-11
> **버전**: 1.0

---

## 1. 프로젝트 비전 및 목표

### 비전
"한 입에 담는 대한민국 여행" — 여행지, 맛집, 특산품, 레시피를 하나의 플랫폼에서 탐색하고, 모든 사용자가 접근 가능한 여행 정보 서비스.

### 목표
- 대한민국의 여행지·맛집·특산품·레시피 정보를 통합 제공
- 반려동물 동반, 휠체어 접근성, 외국어 지원 등 대상별 맞춤 정보 제공
- 한국어/영어 다국어 지원으로 내·외국인 모두 활용 가능
- 공공 데이터(TourAPI, 식약처 레시피)를 활용한 신뢰성 높은 정보 제공
- 고캠핑 API를 활용한 전국 캠핑장 정보 통합 제공
- 날씨·계절 기반 맞춤 여행지 추천

## 4. 사용자 시나리오 (User Stories)

### US-001: 지역별 여행지 탐색
> **As a** 국내 여행자,
> **I want to** 특정 지역의 여행지 목록을 볼 수 있도록,
> **So that** 여행 계획을 세울 수 있다.

**인수 조건:**
- 지역 선택 시 해당 지역의 여행지 목록이 표시됨
- 각 여행지 카드에 대표 이미지, 이름, 간단 설명이 표시됨
- 테마(관광지/문화시설/축제 등) 필터 적용 가능
- 무한 스크롤 또는 페이지네이션 지원

### US-002: 반려동물 동반 장소 검색
> **As a** 반려동물 동반 여행자,
> **I want to** 반려동물 동반 가능한 장소만 필터링할 수 있도록,
> **So that** 반려동물과 함께 방문할 수 있는 곳을 쉽게 찾을 수 있다.

**인수 조건:**
- "반려동물 동반" 필터 토글 제공
- 필터 적용 시 반려동물 동반 가능 장소만 표시
- 각 장소에 반려동물 관련 상세 정보 (크기 제한, 동반 조건 등) 표시

### US-003: 특산품에서 레시피로
> **As a** 미식 탐험가,
> **I want to** 지역 특산품을 보고 관련 레시피를 찾을 수 있도록,
> **So that** 집에서 지역 특산품을 활용한 요리를 해볼 수 있다.

**인수 조건:**
- 특산품 상세 페이지에서 연결된 레시피 목록이 표시됨
- 레시피 페이지에서 단계별 조리법과 재료가 명확히 표시됨
- 레시피에 칼로리, 난이도, 조리 시간 표시

### US-004: 외국인의 한국 여행 탐색
> **As a** 외국인 관광객,
> **I want to** 영어로 한국 여행 정보를 탐색할 수 있도록,
> **So that** 언어 장벽 없이 한국 여행을 계획할 수 있다.

**인수 조건:**
- 영어 UI로 전환 시 모든 인터페이스 텍스트가 영어로 표시
- 여행지명, 주소, 설명이 영어로 제공됨
- 외국인 친화 장소(영어 메뉴, 다국어 직원) 필터 제공
- URL이 `/en/travel` 형태로 영어 로케일 반영

### US-005: 날씨 기반 여행 계획
> **As a** 여행 계획자,
> **I want to** 여행지가 속한 지역의 대표 날씨와 예보를 볼 수 있도록,
> **So that** 날씨에 맞는 여행지를 선택할 수 있다.

**인수 조건:**
- 여행지 상세 페이지에 지역 대표 날씨와 3일 예보 위젯 표시
- 홈페이지에 날씨 기반 추천 여행지 표시
- 계절별 추천 여행지 섹션 제공

### US-006: 휠체어 접근 가능 관광지 검색
> **As a** 휠체어 사용자,
> **I want to** 휠체어 접근 가능한 관광지만 검색할 수 있도록,
> **So that** 이동에 불편함 없이 여행을 즐길 수 있다.

**인수 조건:**
- "휠체어 접근 가능" 필터 토글 제공
- 접근성 배지로 지원 수준 시각적 표시
- 상세 페이지에 구체적 접근성 정보 (경사로, 엘리베이터, 장애인 화장실 등) 표시

### US-007: 지역 허브에서 종합 탐색
> **As a** 여행자,
> **I want to** 특정 지역의 여행지, 맛집, 특산품, 날씨를 한 페이지에서 볼 수 있도록,
> **So that** 지역 전체를 한눈에 파악하고 효율적으로 여행을 계획할 수 있다.

**인수 조건:**
- SVG 한국 지도에서 지역 클릭 시 해당 지역 허브 페이지로 이동
- 허브 페이지에 여행지, 맛집, 특산품, 지역 대표 날씨가 섹션별로 표시
- 각 섹션에서 상세 목록 페이지로 이동 가능

### US-008: 통합 검색
> **As a** 사용자,
> **I want to** 하나의 검색바에서 여행지, 맛집, 특산품을 한번에 검색할 수 있도록,
> **So that** 원하는 정보를 빠르게 찾을 수 있다.

**인수 조건:**
- 홈 히어로 영역과 헤더에 검색바 제공
- 검색 결과가 카테고리별로 그룹화되어 표시
- 검색어 자동완성 또는 추천 검색어 제공 (P2)

### US-009: 캠핑장 탐색
> **As a** 캠핑 여행자,
> **I want to** 지역별·업종별 캠핑장을 검색하고 시설 정보를 비교할 수 있도록,
> **So that** 나에게 맞는 캠핑장을 찾아 캠핑 여행을 계획할 수 있다.

**인수 조건:**
- 지역 선택 시 해당 지역의 캠핑장 목록이 표시됨
- 업종(일반야영장/자동차야영장/카라반/글램핑) 필터 적용 가능
- 부대시설(전기/와이파이/온수/화로대) 필터 적용 가능
- 사이트 바닥 타입(잔디/데크/파쇄석/자갈) 정보 표시
- 캠핑장 카드에 대표 이미지, 이름, 주소, 주요 시설 배지 표시

### US-010: 반려동물 동반 캠핑장 검색
> **As a** 반려동물 동반 캠퍼,
> **I want to** 반려동물 동반 가능한 캠핑장만 필터링할 수 있도록,
> **So that** 반려동물과 함께 캠핑할 수 있는 곳을 쉽게 찾을 수 있다.

**인수 조건:**
- "반려동물 동반 가능" 필터 토글 제공
- 필터 적용 시 반려동물 동반 가능/소형견만 가능 캠핑장 표시
- 각 캠핑장에 반려동물 관련 조건 (크기 제한 등) 표시

## 5. MVP 범위 정의

### MVP에 포함 (Phase 0~4)
- 프로젝트 인프라 셋업 (Next.js, Supabase, Vercel)
- 한국어/영어 다국어 기본 지원
- TourAPI 연동 여행지/맛집 탐색 (목록 + 상세 + 검색)
- 네이버 지도 연동 (위치 표시)
- 특산품/레시피 연동 (COOKRCP01)
- 고캠핑 API 연동 캠핑장 탐색 (목록 + 상세 + 검색 + 필터)
- 대상별 접근성 정보 및 필터
- 날씨 정보 및 지역 허브
- 자체 리뷰/평점 시스템
- 홈페이지 (추천, 검색, 인기 지역)
- SEO 기본 (메타데이터, JSON-LD)

### MVP 이후 (Post-MVP)
- 외부 지도/플레이스 상세 링크 제공 (네이버 지도, 카카오맵)
- 사용자 맞춤 추천 (AI 기반)
- 여행 코스 생성기
- 소셜 기능 (좋아요, 공유, 저장)
- 추가 언어 지원 (일본어, 중국어)
- PWA / 오프라인 지원
- 여행 일정 플래너

- Supabase RLS 정책으로 데이터 접근 제어
- API 키 서버 사이드 관리 (env 변수)

- XSS/CSRF 방어

- WCAG 2.1 AA 준수
- 키보드 네비게이션 지원
- 스크린 리더 호환
- 적절한 색상 대비

### SEO
- Server-side rendering (SSR/SSG)
- 구조화된 데이터 (JSON-LD: Recipe, TouristAttraction, Restaurant, Campground, WebSite)

- 다국어 hreflang 태그
- Google Search Console 등록 + sitemap 제출 + 색인 요청

## 8. 제약 조건 및 가정

### 제약 조건
- TourAPI 일일 호출 제한 (일반 키: 1,000회/일, 활용 키: 10,000회/일)
- 식약처 레시피 API 데이터는 한국어만 제공 (영어 번역 필요)
- 고캠핑 API 데이터는 한국어만 제공 (영어 번역 필요)
- 기상청 API 응답 형식이 복잡하여 파싱 로직 필요
- Supabase 무료 플랜 제한 (500MB DB, 1GB Storage)

### 가정
- 사용자는 모바일(60%)과 데스크톱(40%)을 사용
- TourAPI 데이터는 24시간 캐시로 충분한 신선도 유지
- 초기 MVP에서는 사용자 인증 없이도 탐색 가능 (리뷰 작성 시에만 인증)
- 네이버 지도 및 카카오 API는 MVP 예상 트래픽을 감당할 수 있는 쿼터를 전제로 하며, 실제 운영 전 콘솔 사용량/쿼터를 확인한다
